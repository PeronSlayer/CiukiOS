# 2026-04-19 — OPENGEM-002-BAT: BAT Interpretation Hardening

## Context and goal
Phase 2 of the OpenGEM UX roadmap
([docs/roadmap-opengem-ux.md](../roadmap-opengem-ux.md)). Promote the
in-tree BAT interpreter (`shell_run_batch_file` in
[stage2/src/shell.c](../../stage2/src/shell.c)) from "good enough for
AUTOEXEC.BAT and demo scripts" to a documented, tested, strict subset
of `COMMAND.COM` that can actually run `GEM.BAT`, `SETUP.BAT`, and
generic FreeDOS batch scripts without early abort. No version bump
(baseline `CiukiOS Alpha v0.8.7`). No changes to `main`.

## Decisions made
- **Static-first smoke gate.** Like `test_opengem_smoke.sh` and
  `test_mouse_smoke.sh`, the gate is host-side and static. QEMU runs
  on macOS are unreliable, so the primary contract is asserted by
  greping the stage2 source and the fixture files. A runtime boot-log
  probe is opt-in and degrades to SKIP.
- **Serial-only line echo.** `ECHO ON` prints the expanded line as
  `[ bat ] line: …` through `serial_write()` only; text-console echo
  would duplicate noise with the shell's own output formatting and is
  deferred to a future UX polish (documented in
  `docs/bat-interpreter.md`).
- **Unified IF dispatch.** The pre-existing `IF ERRORLEVEL N GOTO L`
  path was generalized to accept any command after the condition.
  `IF EXIST` and `IF "a"=="b"` use the same fall-through mechanism
  via a `reprocess:` label — the command substring overwrites
  `expanded` and reruns the keyword scan.
- **Frame save/restore is strictly stack-allocated.** No dynamic
  allocation. `saved_argv[10]` lives on the caller frame; nested
  `CALL` into `.BAT` recurses through `shell_execute_line()` which
  re-enters `shell_run_batch_file()`, which saves/restores again.
- **`SHIFT` preserves `%0`.** Real `COMMAND.COM` with `SHIFT /0` can
  shift the batch name; our `SHIFT` always keeps `%0` as the batch
  path to make script-self-location stable across shifts.
- **`%%` is now literal `%`.** Previously the expander couldn't
  produce a literal `%`; this is a strict extension, no behavior
  regression.
- **Scope boundary: no `FOR` / `CHOICE` / pipes / `COMMAND /C` /
  `%*` / delayed expansion.** Explicitly documented as divergences
  in `docs/bat-interpreter.md` so future phases can audit gaps.

## Risks and next step
- **Echo semantics drift from real `COMMAND.COM`.** Our serial-only
  echo is safer but silently different; if a user relies on BAT
  console echo for progress, they will see stage2 shell output
  instead. Tracked in `docs/bat-interpreter.md`.
- **`PAUSE` is a blocking read.** If a BAT reaches `PAUSE` in a
  non-interactive context (e.g. AUTOEXEC.BAT on a CI boot), it will
  halt indefinitely. Mitigation: users avoid `PAUSE` in
  AUTOEXEC-chain scripts; documented as a known gap.
- **`%*` not yet supported.** Scripts that forward "all remaining
  args" as a single string (`EXEC %*`) will silently substitute
  empty. Use `%1 %2 %3 …` + `SHIFT` for now. Planned for a future
  micro-task if any real BAT hits it.
- **Quoting is single-level.** Nested `"` inside `IF "a"=="b"` not
  supported; not observed in real FreeDOS BAT corpus.
