## Decisions Made
1. Next callable DPMI slice chosen: `INT 31h AX=0000h` (Allocate LDT Descriptors). Rationale: it is the first DPMI entry that must return a non-trivial selector value after version query, and its low-3-bits shape (`RPL=3 | TI=1 -> 0x07`) can be validated deterministically.
2. Selector synthesized by stage2 is `0x0087` (arbitrary base 0x80 with the required low-3-bits mask set). This keeps the smoke a contract check, not a real descriptor installer.
3. VGA mode 13h is introduced as a compatibility scaffold only (marker + shell command + static gate). No real mode-13h framebuffer path is wired yet; the real draw path is explicitly deferred to the DOOM graphics milestone so this change does not regress GOP/GUI.
4. BIOS compat markers (`[compat] bios int1Xh baseline ready`) are instrumentation only; they reflect already-established baseline behavior and make DOOM-startup expectations greppable. They do not change runtime behavior.
5. Boot-to-DOOM harness keeps `menu_reached` explicitly `DEFERRED` instead of failing, so the gate is PASS today and becomes a natural target to flip to runtime-verified PASS once a real DOOM runtime path exists.
6. All new runtime tests use the existing static-fallback pattern so the gate stays deterministic even when QEMU cannot run the MZ through FreeDOS within the timeout.

## Risks and Next Step

### Risks
1. The LDT slice still returns a synthetic selector, so a real extender attempting to use it as an actual descriptor would fault. The smoke is intentionally a contract check, not a real allocator.
2. VGA13h is a scaffold; any consumer that expects real ISA VGA ports (`0x3C0..0x3DF`) or a 0xA0000 framebuffer will not work yet.
3. The staged DOOM harness validates packaging-level discovery + source-level markers, not runtime execution beyond packaging.

### Next step (v0.7.1+ candidates)
1. Promote one of the shallow DPMI slices to a real allocator that returns a usable selector (actual LDT table backing), and validate with a binary that touches memory through that selector.
2. Replace the VGA13h scaffold with the first real `mode 13h`-style linear framebuffer draw path (palette setup + plane writes) under a deterministic gate.
3. Introduce a minimal DOS extender regression target that fails deterministically in a known stage so the boot harness moves `menu_reached` from DEFERRED to a real failure-taxonomy classification.
4. Expand `INT 10h` / `INT 16h` beyond marker-only into actual callable bodies used by DOOM startup traces.
