## Technical decisions
1. Decision:
Increase static backbuffer compatibility ceiling to 1024x768 via shared limit constants.
Reason:
Directly satisfies minimum compatibility target requested by user.
Impact:
Video double-buffer path can stay active up to 1024x768 when GOP mode fits.

2. Decision:
Promote 1024x768 ahead of 800x600 in loader preferred fallback list.
Reason:
Ensures mode policy converges to the requested minimum resolution when available.
Impact:
Boot selection behavior becomes aligned with sub-roadmap target.

3. Decision:
Add non-interactive deterministic gate (`test-video-1024`) based on policy and source-of-truth limits.
Reason:
Avoid dependence on interactive runtime behavior while keeping CI-checkable target enforcement.
Impact:
Fast validation signal for future regressions in video compatibility policy.
