## Open Risks Summary

1. **Launch return determinism is not yet guaranteed**: runs can reach launch path without deterministic `returned` marker (`High`).
2. **Memory API stress during OpenGEM payload execution remains a hotspot**: `48h` activity is visible but not yet fully correlated per-call (`High`).
3. **Some high-frequency DOS APIs are only indirectly observed** (`44h`, `4Fh`, `62h`): targeted deeper traces needed for full closure (`Medium`).

## Recommended Follow-up (non-blocking for this scope)

1. Re-run trace with longer timeout and 3-5 labeled samples (`--label`) to improve reproducibility.
2. Correlate acceptance hangs with per-run serial fragments from `build/full/opengem-acceptance-<label>/`.
3. Extend parser to bucket observed `AH` frequencies once richer markers are emitted in logs.
