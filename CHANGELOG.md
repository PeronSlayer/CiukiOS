# Changelog

All notable project-level changes are tracked here.
This changelog is intentionally concise. Every completed task should update `Unreleased` unless the task cuts a release section.

## Unreleased (2026-05-11)

1. Restored the current DOOM/WOLF3D runtime branch around a compact Stage1 MZ/DOS memory layout: primary EXE loading now has a separate MZ load limit, low DOS scratch buffers are isolated from the loader window, MZ tail memory is cleared before handoff, and runtime cache state is reset before external MZ execution.
2. Fixed the DOOM launch regression that stalled at `V_Init: allocate screens`; the dedicated DOOM taxonomy lane now reaches `visual_gameplay=PASS` on the full profile.
3. Fixed WOLF3D black-screen startup in local full-image builds by patching only the injected copy of `WOLF3D.EXE` to bypass the stuck page-flip wait. The source payload under `third_party/WOLF3D` remains untouched, and the final WOLF3D capture is non-black (`720x400`, `colors=6`, `nonblack=24336`).
4. Added SB16 probe/playback evidence through `SB16INIT.COM`, `DRVLOAD.COM /AUDIO`, and QEMU SB16 device wiring. QEMU audio now defaults to on across full, full-CD, taxonomy, and DRVLOAD smoke runners while still allowing `QEMU_AUDIO_MODE=off` for explicit silent runs; the validated path detects the DSP at `0x220`, configures IRQ/DMA mixer state, and completes a real DMA1/IRQ7 SB playback probe. DOOM now launches as `DOOM.EXE`, removing the old `-nosound` and `-nosfx` workarounds while the generated music-only config keeps the unstable SB SFX/DMA path disabled via `snd_channels=0` and `snd_sfxdevice=0`, with the DOOM taxonomy pinned to the ALSA backend to avoid a local PipeWire/OPL QEMU crash, while SB DMA/IRQ sound effects remain follow-up work.
5. Kept the full-CD build inside the Stage1 size budget by making the legacy Stage2 autorun/hardware-validation CD path opt-in instead of forced by default; the active full-CD gate remains the Live/install D: prompt smoke.
6. Restored the README as a complete project entry point, condensed this changelog to release-level facts, and removed temporary local run artifacts while keeping generated build outputs out of the tracked worktree.
7. Improved shell external-command dispatch with GPLv2 FreeCOM-style first-word/rest splitting and `DoExec`-style command-tail packing, so direct `PROGRAM ARG` launches pass the argument tail like the existing `run PROGRAM ARG` path and quoted external command names are parsed without changing prompt text or shell UX; the shared full/full-cd DOS environment now exposes `PATH=C:\APPS;C:\SYSTEM\DRIVERS` for external programs.
8. Relicensed CiukiOS project code from GPLv3 to GPLv2 so GPLv2 FreeDOS/FreeCOM code can be integrated directly, removed the prebuilt FreeCOM binary packaging shortcut, and documented the source-port policy.
9. Added a reversible DOOM audio-profile packaging switch for full/full-cd (`CIUKIOS_DOOM_AUDIO_PROFILE=music-only|sb16`) and a narrow taxonomy pre-launch probe that prints `DEFAULT.CFG` before `run DOOM.EXE`, so runtime investigations can distinguish the intentional music-only bypass from an SB16-enabled launch without changing shell UX or the single-command DOOM run path.
10. Removed the FAT16 shell footer/status strip entirely from Stage1 by turning the footer update/poll path into no-ops and by dropping the remaining decorative horizontal separator lines from the shell chrome, keeping the shell prompt/output UX simpler while recovering Stage1 code budget on both full and full-CD builds.
11. Added `AUDIOTST.COM`, a tiny SB16-focused audio utility that probes the DSP at legacy bases and plays three short DMA-backed test tones with serial/log markers, and packaged it into `\SYSTEM\DRIVERS` for full/full-CD images together with a short `AUDIO.COM` alias for easier invocation from the shell.
12. Fixed bare shell execution of driver helpers from the default `C:\APPS>` prompt by extending Stage1 external-command resolution to probe `\SYSTEM\DRIVERS` in addition to CWD and `\APPS`, so short commands like `AUDIO`, `AUDIOTST`, `SB16INIT`, and `DRVLOAD` no longer require `run ...` or a manual `cd \SYSTEM\DRIVERS`.
13. Expanded `AUDIOKEY.COM` into a more usable interactive SB16 keyboard tester: the arrow keys still hit preset tones, `C D E F G A B` play scale notes, `+` and `-` fine-tune the current pitch in single steps, `SPACE` replays the current note, and the on-screen legend was rewritten to show the controls more clearly without changing the surrounding shell UX.
14. Hardened the DOS child audio environment for legacy SB16-aware apps by exporting a fuller `BLASTER=A220 I7 D1 H5 T6` string from Stage1 instead of the older truncated `A220 I7 D1`, and documented the current DOOM finding that stable SB16 engagement appears to need explicit prelaunch helper priming (`DRVLOAD /AUDIO`) on the real audio-enabled taxonomy lane rather than hidden hardware mutation inside the Stage1 MZ exec path.
15. Added a new `DOOMSB.COM` launcher into `\APPS\DOOM` that copies the bundled SB16 config into both `\APPS\DOOM\DEFAULT.CFG` and `\DOOMDATA\DEFAULT.CFG`, runs `DRVLOAD /AUDIO`, and then launches `DOOM.EXE`, so the current best-known SB16 priming sequence can be invoked as one command without touching the Stage1 MZ exec hot path.
16. Added a short `AKEY.COM` alias for `AUDIOKEY.COM` in `\SYSTEM\DRIVERS`, matching the existing `AUDIO.COM` convenience alias so the interactive keyboard audio tester is faster to launch from the shell.
17. Added quiet-mode support to `SB16INIT.COM` and `DOOMSB.COM`, then ruled out the silent launcher approach for default DOOM startup because the nested helper path and every confirmed SB/DMX SFX path currently stall before visual gameplay; the default DOOM profile remains the stable music-only visual lane while `CIUKIOS_DOOM_AUDIO_PROFILE=sb16-sfx` stays available for focused SFX investigation.
18. Hardened the DOS app visual taxonomy so doubled serial DOOM markers such as `SS__IInniitt` / `HHUU__IInniitt` count as late video evidence, and so a fresh high-diversity screenshot can classify `visual_gameplay` even if the local QEMU audio backend crashes during post-capture shutdown; this keeps runtime stability failures visible without hiding concrete visual evidence.
19. Switched the packaged default DOOM audio profile from silent/music-only SFX bypass to a stable PC speaker SFX fallback (`snd_sfxdevice=1`, `snd_musicdevice=0`, `snd_channels=8`) and wired QEMU full/full-CD/taxonomy runners with `pcspk-audiodev=snd0`, so DOOM has an in-game audio path now while the broken SB/DMX protected-mode SFX branch remains opt-in via `CIUKIOS_DOOM_AUDIO_PROFILE=sb16-sfx`.
20. Changed local QEMU audio backend auto-selection to prefer ALSA before PipeWire for full/full-CD/taxonomy/DRVLOAD lanes, improving the chance that PC speaker DOOM SFX are actually audible instead of only configured.
21. Added narrow serial-only vector tracing for DOS `INT 21h AH=25h/35h` on timer/IRQ vectors `08h`, `0Dh`, `0Fh`, and `1Ch`, so DOOM/DMX protected-mode audio work can distinguish real-mode vector hooks from DOS/4GW internal protected-mode interrupt handling without changing shell UX.
22. Added and fixed a real DOS/4GW protected-mode SB IRQ/DMA probe: `PMIRQSB.COM` launches `DOS4GW.EXE PMIRQSB.LE`, installs DPMI protected-mode interrupt vectors with `INT 31h AX=0204/0205`, checks virtual IRQ enable, verifies timer IRQ reflection, waits on that protected-mode timer long enough for SB16 DMA completion, validates DMA buffers do not cross a 64K boundary, and now covers single-cycle ACK-before-EOI, single-cycle EOI-before-ACK, and auto-init DMA variants without faking any INT31 support in Stage1.
23. Captured a focused DOOM DMX/SB16 reverse handoff with the exact local `DOOM.EXE` identity, embedded DOS/4G LE payload offset, public PCDoom/Doom Vanille/Apogee runtime map, observed DOOM startup markers, and the narrowed next step: a DMX-like protected-mode timer/task/FX-DMA probe in `PMIRQSB` before touching Stage1 or guessing at DOOM config again.
24. Extended `PMIRQSB` with an opt-in `TASK` mode that installs DOS/4GW protected-mode timer/SB IRQ vectors and dispatches repeated timer-paced single-cycle DMA playbacks with `[PMIRQSB] TASK HIT`, `FXDMA START`, `FXDMA IRQ`, and `TASK PASS` markers, targeting the DMX `TSM_NewService`/`WAV_PlayMode` style failure surface without changing Stage1 or user-facing shell UX.
